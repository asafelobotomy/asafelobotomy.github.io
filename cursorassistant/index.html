cursorAssistant

Lifecycle management for Cursor AI surfaces in any workspace: managed installs, lockfile drift detection, profiles and packs, and specialist routing via .cursor/ and AGENTS.md.

Quick start: open the setup page and click Install in Cursor. That bootstraps the global package and MCP only — the project interview runs inside Cursor after you reload the window.

Open setup page

Setup flow

  1. Install in Cursor On the setup page, approve the MCP install dialog. The server uses a pinned git tag (review the command in Cursor before confirming).
  2. Open your project File → Open Folder for the workspace you want to configure.
  3. Reload Window Developer → Reload Window so plugin skills and commands load.
  4. Run the interview In chat: Set up cursorAssistant in this workspace, /cursor-assistant:setup-workspace, or MCP tool lifecycle_configure. This writes .cursor/ and the lockfile.

Terminal install

Full bootstrap plus interview in one step:

curl -fsSL https://raw.githubusercontent.com/asafelobotomy/cursorassistant/v0.12.0/scripts/install-from-github.sh | bash -s -- .

Bootstrap only (complete setup in Cursor afterward):

curl -fsSL https://raw.githubusercontent.com/asafelobotomy/cursorassistant/v0.12.0/scripts/bootstrap-from-github.sh | bash
